Cheque Requisition System — Complete Application Prompt
Build a Cheque Requisition System for Liverpool Victoria (LV=). This is a React + Express + PostgreSQL application using Vite, TypeScript, Tailwind CSS v4, Shadcn/UI (Radix primitives), and Wouter routing. The primary route / is the main requisition viewer/editor. The application is a frontend-driven prototype with mock data — no backend persistence is used at runtime.

INTERACTION FLOWS
Navigate records: First/Prev/Next/Last buttons update currentIndex (0-based) in state, counter shows currentIndex + 1 of data.length
Edit flow: Click Edit → if drawn/processed shows ErrorDialog → else enters edit mode, Edit button becomes Save (brand variant), header shows OK/Cancel buttons, form fields become PrimaryInputs
Save flow: OK or Save → merges editData into data array → shows InformationDialog "Requisition updated successfully"
New flow: Click New → opens NewRequisitionPanel dialog → fill form → Create → validates Acc No (error if empty) → asks confidential → creates record → toast notification
Find flow: Type in "Find Req" → filters dropdown → click result OR click Find button with no search → opens FindRequisitionPanel dialog
Print flow: Click Print → if requestedBy === 'WQU' → ErrorDialog "Queue Print Error: No User profile exists - please contact IT" → else → react-to-print prints content + InformationDialog "Your cheque rec has been submitted to the Print Queue and will be printed shortly..."
Accounts flow: Click Accts → opens AccountsDialog with current record data
Codes flow: Click Codes → opens CodesDialog for code management
Logout flow: Click Logout → QueryDialog "Are you sure you want to log out?" → Yes logs out, No dismisses
